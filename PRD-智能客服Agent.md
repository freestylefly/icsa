## 1. 产品概述

### 1.1 产品愿景

打造一款**开箱即用的智能客服 SaaS 平台**，让中小企业以极低的成本获得企业级的 7×24 小时智能客服能力，将人工客服从重复性问题中解放出来，专注于高价值的客户服务工作。

### 1.2 产品定位

| 维度 | 描述 |
|------|------|
| 产品类型 | SaaS 平台 - 多租户架构 |
| 目标市场 | 中小企业（电商 + SaaS 公司） |
| 核心价值 | 降低人工成本 + 7×24 在线 + 提升响应效率 |
| 技术路线 | 基于现有大模型 API + 自研知识库引擎 |
| 商业模式 | 订阅制（按坐席/按月付费） |

### 1.3 价值主张

**对企业主：**
- 降低 70% 以上客服人力成本
- 无需技术团队，30 分钟完成部署
- 按使用量付费，无前期投入

**对客服团队：**
- 自动处理 80% 重复性问题
- 智能辅助人工回复，提升效率
- 7×24 小时不间断服务

**对终端用户：**
- 秒级响应，无需等待
- 问题一次解决率提升 50%
- 一致的服务质量体验

### 1.4 竞品分析

| 功能维度 | 阿里云小蜜 | Intercom | 本产品差异化 |
|----------|-----------|----------|-------------|
| 部署方式 | 需技术对接 | SaaS | **零代码配置** |
| 价格门槛 | 高（企业级） | 高（$74/月） | **低（¥299/月起）** |
| 知识库构建 | 手动配置 | 手动配置 | **AI 自动学习** |
| 多租户支持 | 是 | 是 | 是 |
| 中文支持 | 优秀 | 一般 | **优秀** |
| 定制化程度 | 高 | 中 | **中（MVP 阶段）** |

## 4. 功能详细说明

### 4.1 知识库管理

#### 4.1.1 文档上传与解析（P0）

**需求描述：**
- 支持上传格式：PDF、Word（.doc/.docx）、TXT、Markdown
- 单文件大小限制：≤10MB
- 自动解析文档内容，按段落/标题切分为知识片段
- 支持预览解析结果，手动调整切分
- 解析后的知识片段自动向量化存储

**验收标准：**
- 上传 100 页 PDF 文档，解析时间 ≤30 秒
- 知识片段切分准确率 ≥85%
- 支持批量上传（≤50 个文件/次）

#### 4.1.2 知识库问答对管理（P0）

**需求描述：**
- 支持手动添加问答对（问题 + 答案）
- 支持一问多答（同一问题多个答案变体）
- 支持问答对分类/标签管理
- 支持问答对导入/导出（Excel 格式）
- 支持问答对搜索与批量编辑

**验收标准：**
- 添加问答对响应时间 ≤1 秒
- 支持至少 10000 条问答对/租户
- Excel 导入 1000 条记录时间 ≤30 秒

#### 4.1.3 智能知识抽取（P1）

**需求描述：**
- 自动分析历史对话记录
- 识别高频问题与优质回答
- 一键将对话转为知识条目
- 支持人工审核后入库

**验收标准：**
- 知识抽取准确率 ≥70%
- 支持按时间范围筛选对话

### 4.2 对话引擎

#### 4.2.1 多轮对话管理（P0）

**需求描述：**
- 支持上下文记忆（默认保留最近 10 轮对话）
- 支持指代消解（如"它"、"这个"等代词理解）
- 支持对话状态追踪
- 支持主动追问澄清

**验收标准：**
- 上下文理解准确率 ≥80%
- 对话响应时间 ≤2 秒（P95）

#### 4.2.2 意图识别（P0）

**需求描述：**
- 基于大模型 API 进行意图分类
- 支持自定义意图类别
- 支持意图置信度输出
- 支持未知意图识别

**验收标准：**
- 常见意图识别准确率 ≥90%
- 支持至少 50 个意图类别/租户

#### 4.2.3 置信度阈值（P0）

**需求描述：**
- 可配置置信度阈值（默认 0.7）
- 低于阈值时自动转人工或回复"我不确定"
- 支持按意图类型设置不同阈值
- 支持阈值调整后的效果对比

**验收标准：**
- 阈值配置实时生效
- 提供阈值调整建议（基于历史数据）

### 4.3 渠道集成

#### 4.3.1 Web 聊天 widget（P0）

**需求描述：**
- 提供 JavaScript SDK，一行代码嵌入网站
- 支持自定义聊天窗口样式（颜色、位置、Logo）
- 支持移动端适配
- 支持聊天窗口打开/关闭状态记忆
- 支持未读消息角标提示

**验收标准：**
- SDK 加载时间 ≤500ms
- 兼容主流浏览器（Chrome、Safari、Firefox、Edge）
- 移动端适配良好（iOS/Android 主流浏览器）

#### 4.3.2 微信公众号接入（P1）

**需求描述：**
- 支持公众号自动回复（关注后、关键词、默认）
- 支持公众号菜单配置
- 支持图文消息推送
- 支持微信用户信息同步

**验收标准：**
- 消息响应时间 ≤3 秒
- 符合微信公众号 API 规范

### 4.4 人工协作

#### 4.4.1 人机协作转接（P0）

**需求描述：**
- 支持用户主动请求转人工
- 支持机器人自动判断转人工（低置信度/敏感问题）
- 转接时自动传递对话历史给人工坐席
- 支持转接排队与通知
- 支持人工坐席上线/下线状态

**验收标准：**
- 转接响应时间 ≤5 秒
- 对话历史完整传递
- 支持排队人数提示

#### 4.4.2 对话历史记录（P0）

**需求描述：**
- 存储所有对话记录（机器人 + 人工）
- 支持按时间/用户/坐席筛选
- 支持对话详情查看（含时间线）
- 支持对话导出（Excel/PDF）
- 数据保留期限：至少 180 天

**验收标准：**
- 历史记录查询响应时间 ≤2 秒
- 支持至少 100 万条记录/租户

### 4.5 数据分析

#### 4.5.1 对话量统计（P0）

**需求描述：**
- 日/周/月对话量趋势图
- 按渠道来源统计（Web/微信/企业微信）
- 按时间段统计（24 小时分布）
- 支持数据导出

**验收标准：**
- 数据 T+1 更新（次日可见前一天完整数据）
- 图表加载时间 ≤3 秒

#### 4.5.2 问题解决率（P0）

**需求描述：**
- 自动统计机器人独立解决问题比例
- 统计转人工率
- 统计用户重复咨询率
- 支持按问题类型分析解决率

**验收标准：**
- 解决率计算逻辑透明可解释
- 数据准确率 ≥95%

### 4.6 系统管理

#### 4.6.1 多租户管理（P0）

**需求描述：**
- 租户数据完全隔离（数据库/存储/配置）
- 支持租户独立域名/子域名访问
- 支持租户级配置管理
- 支持平台级租户管理后台

**验收标准：**
- 租户间数据零泄露
- 支持至少 1000 个并发租户

#### 4.6.2 计费与订阅（P0）

**需求描述：**
- 支持多套餐配置（基础版/专业版/企业版）
- 支持按月/按年订阅
- 支持自动续费
- 支持使用量超额提醒
- 支持发票管理

**验收标准：**
- 订阅状态实时同步
- 支付接口对接稳定（支付宝/微信）

## 5. 非功能性需求

### 5.1 性能需求

| 指标 | 要求 | 备注 |
|------|------|------|
| **对话响应时间** | P95 ≤2 秒 | 从用户发送到机器人回复 |
| **页面加载时间** | ≤3 秒 | 管理后台首屏加载 |
| **API 响应时间** | P95 ≤500ms | 除对话接口外的其他 API |
| **并发对话数** | ≥1000 路/租户 | 单租户最大并发 |
| **系统吞吐量** | ≥10000 请求/秒 | 平台整体 |

### 5.2 安全需求

| 安全维度 | 要求 |
|----------|------|
| **数据传输** | 全站 HTTPS，TLS 1.3 |
| **数据存储** | 敏感数据加密存储（AES-256） |
| **租户隔离** | 数据库级隔离 + 应用级权限控制 |
| **访问控制** | 支持 RBAC 权限模型，支持 MFA |
| **审计日志** | 记录所有敏感操作（登录、配置变更、数据导出） |
| **合规要求** | 符合《网络安全法》《数据安全法》要求 |
| **漏洞管理** | 定期安全扫描，高危漏洞 48 小时内修复 |

### 5.3 可用性需求

| 指标 | 要求 | 备注 |
|------|------|------|
| **系统可用性** | ≥99.9% | 月度可用性 |
| **故障恢复时间** | RTO ≤30 分钟 | 重大故障恢复目标 |
| **数据恢复点** | RPO ≤5 分钟 | 数据丢失容忍度 |
| **备份策略** | 每日全量备份 + 每小时增量备份 | 保留 30 天 |
| **灾备方案** | 支持跨区域灾备（P1 阶段实现） |

### 5.4 可扩展性需求

- 支持水平扩展，单集群支持≥10 万并发用户
- 支持插件化架构，便于后续功能扩展
- 支持多区域部署，便于国际化扩展

### 5.5 兼容性需求

| 类型 | 要求 |
|------|------|
| **浏览器** | Chrome 90+、Safari 14+、Firefox 88+、Edge 90+ |
| **移动端** | iOS 14+、Android 10+ 主流浏览器 |
| **API 版本** | 支持 API 版本管理，向后兼容至少 2 个大版本 |

## 6. MVP 范围定义

### 6.1 MVP 目标

**时间：** 2 个月（8 周）  
**团队：** 1-2 名全栈开发  
**核心目标：** 验证产品核心价值，获取首批付费用户

### 6.2 MVP 功能范围（P0 子集）

| 模块 | MVP 包含功能 | 简化说明 |
|------|-------------|----------|
| **知识库管理** | 文档上传与解析、问答对管理 | 仅支持 TXT/Markdown 格式，Excel 导入简化 |
| **对话引擎** | 多轮对话、意图识别、置信度阈值 | 使用现有大模型 API，不自研模型 |
| **渠道集成** | Web 聊天 widget | 仅 Web 端，样式可配置项简化 |
| **人工协作** | 人机转接、对话历史记录 | 人工坐席管理简化（上线/下线即可） |
| **数据分析** | 对话量统计、问题解决率 | 基础报表，支持导出 |
| **系统管理** | 多租户、权限管理、计费订阅 | 支持 2-3 个套餐，支付对接简化 |

### 6.3 MVP 不包含功能（P1/P2）

- 微信公众号/企业微信接入
- 智能知识抽取
- 情感识别
- 智能辅助回复
- 用户满意度评价
- 热点问题分析
- 自定义报表
- 品牌定制
- API 开放接口

### 6.4 MVP 技术方案

| 技术选型 | 方案 | 理由 |
|----------|------|------|
| **前端框架** | React + Ant Design | 开发效率高，组件丰富 |
| **后端框架** | Node.js + Express | 全栈统一语言，开发速度快 |
| **数据库** | PostgreSQL + pgvector | 支持向量检索，开源免费 |
| **大模型 API** | 通义千问/文心一言 | 中文能力强，成本可控 |
| **部署方式** | Docker + 云服务器 | 快速部署，成本低 |
| **即时通讯** | Socket.IO | 实时对话，成熟稳定 |

### 6.5 MVP 开发计划

| 阶段 | 时间 | 交付内容 |
|------|------|----------|
| **Phase 1** | 第 1-2 周 | 基础架构搭建、多租户系统、知识库管理 |
| **Phase 2** | 第 3-4 周 | 对话引擎开发、Web widget 开发 |
| **Phase 3** | 第 5-6 周 | 人工协作功能、数据分析、计费系统 |
| **Phase 4** | 第 7-8 周 | 联调测试、Bug 修复、上线准备 |

### 6.6 MVP 成功标准

| 指标 | 目标值 |
|------|--------|
| **付费用户数** | ≥10 家 |
| **月活租户** | ≥20 家 |
| **日均对话量** | ≥1000 条 |
| **问题解决率** | ≥75% |
| **用户满意度** | ≥4.0/5.0 |
| **系统可用性** | ≥99% |
