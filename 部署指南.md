## 🚀 快速部署（Docker Compose - 推荐）

### 步骤 1: 准备项目

```bash
# 进入项目目录
cd /Users/canghe/lobsterai/project

# 确认项目结构
ls -la
# 应该看到：apps/, docker/, docs/, scripts/ 等目录
```

### 步骤 2: 配置环境变量

```bash
# 进入 docker 目录
cd docker

# 复制环境变量模板
cp .env.example .env

# 编辑 .env 文件，修改以下关键配置：
# - POSTGRES_PASSWORD: 设置强密码
# - REDIS_PASSWORD: 设置强密码
# - MINIO_SECRET_KEY: 设置强密码
# - JWT_SECRET: 生成随机密钥（建议 32+ 字符）
# - ALIYUN_API_KEY: 填入阿里云 API 密钥
# - FRONTEND_URL: 你的域名（如 https://cs.yourcompany.com）
```

**.env 关键配置示例：**

```bash
# 数据库配置
POSTGRES_USER=icsa
POSTGRES_PASSWORD=YourStrongPassword123!
POSTGRES_DB=icsa

# Redis 配置
REDIS_PASSWORD=YourRedisPassword456!

# MinIO 配置
MINIO_ACCESS_KEY=minioadmin
MINIO_SECRET_KEY=YourMinioPassword789!

# JWT 配置
JWT_SECRET=VeryLongRandomSecretKeyForJWTTokenSigning123456

# 阿里云 API 配置
ALIYUN_API_KEY=your_actual_api_key_here

# 域名配置
FRONTEND_URL=https://cs.yourcompany.com
REACT_APP_API_URL=https://api.yourcompany.com
```

### 步骤 3: 启动所有服务

```bash
# 在 docker 目录下执行
cd /Users/canghe/lobsterai/project/docker

# 启动所有服务（生产环境）
docker compose -f docker-compose.prod.yml up -d

# 查看服务状态
docker compose -f docker-compose.prod.yml ps

# 查看启动日志
docker compose -f docker-compose.prod.yml logs -f
```

**预期输出：**

```
NAME                STATUS              PORTS
icsa-postgres       Up (healthy)        5432/tcp
icsa-redis          Up (healthy)        6379/tcp
icsa-minio          Up (healthy)        9000/tcp, 9001/tcp
icsa-backend        Up (healthy)        3001/tcp
icsa-frontend       Up (healthy)        3000/tcp
```

### 步骤 4: 数据库初始化

```bash
# 执行数据库迁移
docker exec icsa-backend pnpm db:generate
docker exec icsa-backend pnpm db:migrate

# 创建初始管理员账户
docker exec icsa-backend pnpm db:seed
```

### 步骤 5: 验证部署

```bash
# 检查后端健康状态
curl http://localhost:3001/health

# 预期输出：
# {"status":"ok","timestamp":"2026-04-16T12:00:00.000Z","services":{"database":"ok","redis":"ok","minio":"ok"}}

# 检查前端
curl http://localhost:3000

# 检查 MinIO
curl http://localhost:9001/minio/health/live

# 检查 Redis
docker exec icsa-redis redis-cli ping
# 预期输出：PONG

# 检查 PostgreSQL
docker exec icsa-postgres pg_isready
# 预期输出：/tmp:5432 - accepting connections
```

### 步骤 6: 访问应用

打开浏览器访问：

| 服务 | 地址 | 说明 |
|------|------|------|
| 前端 | http://localhost:3000 | 管理后台 |

**默认管理员账户：**

## 🔧 高级配置

### 配置 HTTPS（生产环境必需）

#### 方式一：使用 Nginx 反向代理

```bash
# 1. 准备 SSL 证书
mkdir -p /Users/canghe/lobsterai/project/docker/ssl
cp /path/to/your/cert.crt docker/ssl/
cp /path/to/your/private.key docker/ssl/

# 2. 修改 docker-compose.prod.yml，启用 nginx 服务

# 3. 重启服务
docker compose -f docker-compose.prod.yml up -d nginx

# 4. 验证 HTTPS
curl -k https://your-domain.com/health
```

#### 方式二：使用 Let's Encrypt 免费证书

```bash
# 使用 certbot 获取证书
docker run --rm -v /etc/letsencrypt:/etc/letsencrypt \
  -v /var/lib/letsencrypt:/var/lib/letsencrypt \
  certbot/certbot certonly --standalone \
  -d your-domain.com -d api.your-domain.com
```

### 配置域名解析

在 DNS 服务商处添加记录：

```
# A 记录
cs.yourcompany.com    →  你的服务器 IP
api.yourcompany.com   →  你的服务器 IP

# 或者 CNAME 记录
cs    →  your-domain.com
api   →  your-domain.com
```

## 🛡️ 安全加固

### 1. 修改默认密码

```bash
# 修改 .env 文件中的所有默认密码
# - POSTGRES_PASSWORD
# - REDIS_PASSWORD
# - MINIO_SECRET_KEY
# - JWT_SECRET

# 重启服务使配置生效
docker compose -f docker-compose.prod.yml down
docker compose -f docker-compose.prod.yml up -d
```

### 2. 配置防火墙

```bash
# macOS (使用 pfctl)
# 创建 /etc/pf.conf 规则，只开放必要端口

# 或者使用云服务商的安全组
# - 开放 80 (HTTP)
# - 开放 443 (HTTPS)
# - 限制 3001, 5432, 6379, 9000 仅内网访问
```

### 3. 启用 API 限流

在 `apps/backend/src/middleware/rate-limiter.ts` 中配置：

```typescript
// 默认配置：每 IP 每分钟 100 次请求
const limiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 分钟
  max: 100, // 每个 IP 最多 100 次请求
});
```

### 4. 启用日志审计

```bash
# 查看审计日志
docker compose -f docker-compose.prod.yml logs backend | grep "AUDIT"
```

## 🐛 故障排查

### 常见问题 1: 后端启动失败

```bash
# 查看详细错误
docker logs icsa-backend

# 检查数据库连接
docker exec icsa-backend ping -c 3 postgres

# 检查环境变量
docker exec icsa-backend env | grep DATABASE

# 常见原因：
# - 数据库未启动
# - 密码错误
# - 端口被占用
```

### 常见问题 2: 数据库迁移失败

```bash
# 查看迁移错误
docker exec icsa-backend pnpm db:migrate

# 重置数据库（谨慎使用！）
docker exec icsa-backend pnpm db:reset

# 重新迁移
docker exec icsa-backend pnpm db:migrate
```

### 常见问题 3: WebSocket 连接失败

```bash
# 检查 Nginx 配置（如果使用）
# 确保包含以下配置：

location /socket.io/ {
  proxy_pass http://backend:3001;
  proxy_http_version 1.1;
  proxy_set_header Upgrade $http_upgrade;
  proxy_set_header Connection "upgrade";
}

# 重启 Nginx
docker compose -f docker-compose.prod.yml restart nginx
```

### 常见问题 4: 内存不足

```bash
# 查看容器内存使用
docker stats

# 调整 Docker 资源限制（docker-compose.prod.yml）
services:
  backend:
    deploy:
      resources:
        limits:
          memory: 2G
        reservations:
          memory: 512M
```
